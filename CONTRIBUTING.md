# 🤝 Contributing to ElectIQ

Thank you for helping make civic education more accessible!

## How to Contribute

### Reporting Bugs
Open a GitHub Issue with:
- Steps to reproduce
- Expected vs actual behaviour
- Browser and OS version

### Suggesting Features
Open a GitHub Issue tagged `enhancement` describing the feature and why it would help users.

### Submitting a Pull Request
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test in Chrome, Firefox, and Safari
5. Open a pull request describing your changes

## Code Style

- Use 2-space indentation
- Write semantic, accessible HTML
- Add comments for non-obvious logic
- Keep CSS variables in `:root {}` for easy theming
- Prefer CSS animations over JS for visual effects

## Content Guidelines

ElectIQ is **strictly nonpartisan**. All contributions must:
- Present information factually without political bias
- Avoid implying any party, candidate, or ideology is better or worse
- Cite authoritative civic sources (vote.gov, FEC.gov, NCSL.org)
- Be reviewed for accuracy by at least one other contributor

## Commit Messages

Use conventional commits:
```
feat: add state-specific registration deadlines
fix: quiz score not resetting on replay
docs: update SETUP.md with Vercel instructions
style: improve mobile layout for timeline cards
```
